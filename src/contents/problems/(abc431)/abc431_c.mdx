## 問い

重さが各 $H_i$ の $N$ 個の<Color c="#c07">頭パーツ</Color>と、
重さが各 $B_i$ の $M$ 個の<Color c="#04d">体パーツ</Color>がある。\
ある<Color c="#c07">頭パーツ</Color>と、
それより重い<Color c="#04d">体パーツ</Color>の組合せで $1$ 体のロボットが作れるとき、\
$K$ 体以上のロボットが作れるか判定せよ。

## 解く

- 貪欲
- いつもしっくり来ないまま時間かけて適当にACしてしまうので、\
読んだだけでピンと来るようにしたい
- 頭パーツと体パーツで対称性はある
- :o: <Color c="#c07">頭パーツ</Color>を**重い順**で見て、組み合わせられる<Color c="#04d">体パーツ</Color>があるなら\
そのうち一番重いものを選んでロボットを作る
- :o: <Color c="#04d">体パーツ</Color>を**軽い順**で見て、組み合わせられる<Color c="#c07">頭パーツ</Color>があるなら\
そのうち一番軽いものを選んでロボットを作る
- :x: <Color c="#04d">体パーツ</Color>を<Red>**重い順**</Red>で見て、組み合わせられる<Color c="#c07">頭パーツ</Color>があるなら\
そのうち一番重いものを選んでロボットを作る
- 軽い頭パーツに比べて、重い頭パーツは相方の体パーツの選択肢が少ない
  - <Faint>重い体パーツに比べて、軽い体パーツは <Size s="14">以下略</Size></Faint>
  - 「選択肢が少ない」というのは、単に数の話じゃなくて、\
  重い頭パーツに与えられた選択肢は全て軽い頭パーツでも選ぶことができて、\
  さらに軽い頭パーツには追加の選択肢がある可能性がある、という意味
  - ある体パーツをどちらの頭パーツの相方にするか決めるとき、\
  「軽い頭パーツにはまだ余裕があるが、重い頭パーツはもう後がない」\
  という状況はありうるが、その逆は起こり得ない
  - したがって、常に重い頭パーツ側を優先させてよい
  - というところまでをすんなりと理解できればよいのですがぁ
  - 「選択肢の少ないほうを優先」をパターンとして覚えてしまったほうがよい？

## 書く

- 重い頭パーツを順に見る
  - その時点で残っている一番重い体パーツを見る
  - ロボットにできるなら、その体パーツを使ってロボットを作る
  - 一番重い体パーツでもダメなら、\
  その頭パーツはもう使える見込みがないので飛ばす

```cpp
void solve() {
    ll cin(n,m,k);
    vll H(n),B(m);cin>>H>>B;
    sort(rall(H));
    sort(rall(B));
    ll j=0;
    for(auto&h:H){
        if(j<m&&B[j]>=h){
            j++;
        }
    }
    ansYn(j>=k);
}
```
